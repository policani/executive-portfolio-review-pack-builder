# Handoff Rules

## To PMO Governance Operations Log

Send decisions, actions, risks, unresolved items, owner commitments, due dates, escalation notes, value follow-up actions, validation needs, and next-cycle carry-forward topics.

Do not send speculative decisions or AI-inferred approvals.

## To Portfolio Capacity Sequencing Planner

Send capacity constraints, sequencing conflicts, shared-team bottlenecks, cross-initiative dependencies, funding constraints, and delivery tradeoff questions.

Do not send routine status or items that need only note-taking.

## To Business Case, Charter, or Scoring modules

Send back only when the input exposes missing investment logic, missing authorization boundaries, or scoring/prioritization gaps.

## To Value Realization Governance Ledger

Send supported, maturing, at-risk, unsupported, or finance-validation-needed value signals when the executive review changes the follow-up route or exposes a measurement gap. Include the measurement contract fields that are known: expected outcome, benefit type, metric, baseline, target, actual if available, measurement period, source, measure owner, review cadence, validation need, confidence, realization risk, finance-sensitive flag, and downstream route.

Do not send executive discussion notes as certified ROI, savings, risk reduction, or realized benefits unless the user supplied explicit human validation.
